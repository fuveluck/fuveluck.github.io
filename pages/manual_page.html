How to use the First-order Logic Horn Fragment to Prolog Converter

Overview

This converter allows you to input logical expressions in first-order logic and convert them first into CNF clauses and then, if it is a Horn fragment, into Prolog syntax. It also supports LaTeX rendering for easy input of formulas. At the same time, it places limitations on the use of special symbols. This project is a part of my thesis.

Getting Started

Follow these steps to use the converter::

  • Navigate to the input area, where you can type or insert logic symbols using the available buttons. Be careful, as copying symbols in an unacceptable format can end with errors.
  • Click the "Convert" button to transform your expression into Prolog syntax.
  • View the Prolog output in the result area below the input field. If you're not sure about the answer, you can check steps in the "CNF" page where you can find the logic of conversion to CNF format for your input FOL formula. Be careful with parentheses as they influence the logic of transformation (better more than less).
  • You can copy your output with a button in output area. If needed, click "Clear" to reset the input field.

Input Field

Special conditions for your formula to convert successfully:

  • \Rightarrow Implication (rendered as \Rightarrow)
  • \land Conjunction (rendered as \land)
  • \lor Disjunction (rendered as \lor)
  • \neg Negation (rendered as \neg)
  • \exists Existential quantifier (rendered as \exists)
  • \forall Universal quantifier (rendered as \forall)
  • \top Top (rendered as \top)
  • \bot Bottom (rendered as \bot)
  • Parentheses ( ) for grouping terms
  • /^[A-Z][0-9]*$/ Predicate symbols, to represent relations between objects you can use only uppercase letters with numbers after them. It is really important to use parentheses after predicates to initialize its terms, for example: {X, P1}
  • /^[a-z][0-9]*$/ Function symbols, to represent specific objects in the domain that depend on others, you can use only lowercase letters with numbers after them. It is really important to use parentheses after functions to initialize its terms, for example: {f, a1}
  • /^[a-r][0-9]*$/ Constant symbols, to represent specific objects of domain you can use only lowercase letters from "a" to "r" with numbers after them, for example: {b, a1}
  • /^[s-z][0-9]*$/ Variable symbols, to represent unknown objects of domain you can use only lowercase letters from "s" to "z" with numbers after them, for example: {v, z1}
  • To represent quantified variables you cannot use parentheses before predicates, for example: {\left(\forall x P(x)\right)}
  • Since the project uses MathLive input field to go to a new line you will need to use special construction (if you would like to paste multiple formulas) or just use Ctrl+Enter<\code>, for example: \displaylines{P(a,b)\\ O(a)}

Examples of input formulae you can find on the main page in navigation bar :)

Output Field

The Prolog output field will display the result in Prolog syntax after clicking "Convert." You can copy the output by clicking the copy button next to the output area and try it out online, for example here Prolog. You can paste all facts and rules into the program, but queries need to be pasted without the "-?" into another window.
